kwbackup

kwbackup ist ein simples Script welches Verzeichnisse ihrer Wahl und das OS backupdatet. Man kann dieses auch per crontab Eintrag erledigen/konfigurieren.

Notiz: Programme sind tar und bzip2.

2 Backup

3 System_Backup


1 Konfiguration

Backup_Verzeichnis: Verzeichnis in der die Backups gespeichert werden sollen. Es werden darunter die Verzeichnisse Rechnername/{sys,dirs} angelegt.

Mount_Verzeichnis: Falls die Backups auf einer lokalen/entfernten Partition/Verzeichnis, die nicht ständig ins Dateisystem eingebunden sind, gespeichert werden, kann man dieses Verzeichnis hier einstellen. Das Verzeichnis/Partition und der Mountpunkt müssen in der /etc/fstab stehen.

Verzeichnisse: Eingabe von 1 oder mehreren Verzeichnissen, getrennt durch 1 Leerzeichen, von denen ein Backup angelegt werden soll.

crontab: Automatisierung, der Konfiguration, durch cron. Es wird dann das Script kwbackup_cron zu einer bestimmten Zeit ausgeführt.


2 Backup

Backup der konfigurierten Verzeichnisse. Sie benötigen dazu Schreibzugriffe auf das Backup_Verzeichnis/Rechnername/dirs. Die Rechte kann man noch nicht einstellen.

Wenn Backups älter als 30 Tage sind, werden komplette Backups angelegt und die alten gelöscht. Ansonsten werden alle Dateien die (sich) in den letzten 7 Tagen angelegt (verändert) wurden gesichert.

Notiz: Backups werden durchnummeriert, falls man den Prozess wiederholt.


3 System_Backup

Backup des Systems. Wird nur angezeigt, wenn man als root eingeloggt ist. Die /home Verzeichnisse zählen standardmässig nicht dazu.

Wenn Backups älter als 60 Tage sind, werden komplette Backups angelegt und die alten gelöscht. Ansonsten werden alle Dateien die (sich) in den letzten 7 Tagen angelegt (verändert) wurden gesichert.

Notiz: Backups werden durchnummeriert, falls man den Prozess wiederholt.
